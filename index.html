Exploring demographics of ideal jobs

By plotting data using an overlapping area chart with opacity for each dataset, we can visually see how skills across jobs overlap while exploring demand.

Search Results by Keyword LinkedIn

The most in demand job from our team's ideal jobs, as confirmed by LinkedIn search results, is an IT consultant, coming in at 694 results from a total of 1386 results (50%).

In second place is full stack developer, coming in at 574 results (41%).

In last place is junior developer, coming in at 118 results (8%).

This supports the idea that the demand for senior developers far exceeds the demand for junior developers; likewise, the jack-of-all trades consultant's skillset isn't limited to working in software alone, so demand is high for good reason.

This trend is mirrored in the data provided by Burning Glass Technologies in that developer ranks lower than fullstack developer, which ranks lower than senior developer, and at the very top of their chart is a job title solutions architect (Top IT Job Titles March, 2018), which is a fancy term for a consultant.

    Machine Learning

    Let's start with defining what is meant by machine learning because the layperson can incorrectly use AI and machine learning as synonyms. Arthur Samuel, famous for creating the first self-learning program that allowed players to play checkers against a computer, coined the term “machine learning” in 1951 (En.wikipedia.org, 2019). Machine learning is what allowed Arthur's computer opponent to make decisions based on the moves of a human player—the computer opponent actually improved over time; in other words, machine learning is a big part of AI, but not AI itself.

    “Machine learning is, at its core, the process of granting a machine or model access to data and letting it learn for itself (Valchanov, 2019).” This is perhaps too vague a statement to shed a great deal of light, so let's go back 15 years earlier before Arthur's conceptualisation of a self-learning machine. In 1936, Alan Turing, famous for breaking the Enigma Code and philosophising the Turing Test (En.wikipedia.org, 2019), published a paper called On Computational Numbers. This paper spoke of a Universal Machine that could make computations. And this was one of the first modern moments of data mining.

    Data mining gave birth to machine learning, but data alone is useless without interpretation.

    Imagine we have a data set containing 2 million social media accounts who use a profile picture that predominantly makes use of the colour blue. A data scientist could look at this mass of mined data and make predictions based on users who prefer to feature this blue in their profile pictures—let's say people with blue profile pictures on average complain of sleep problems more than the general population—but wouldn't it be nice if both the mining of this data and its interpretation could all be automated.

    That way, we could draw conclusions on the sleep health of anyone based on colour preference without having to write multiple versions of the same program for each colour, and most importantly, without having to collect another 2 million social media accounts per colour.

    Better still, wouldn't it be amazing if the conclusions this program draws happen to change overtime according to the data inputted? Say, just like a computer getting better at checkers the more opponents it plays against. It's infinitely more efficient to feed data into a machine over time than requiring a massive data set before meaningful results can be obtained, and this is a massive advantage of a self-learning algorithm.

    In 2019, we can do a great deal more than mine data and lose at checkers. One of the most impressive implementations of machine learning is NVIDIA's artificial faces program (The Verge, 2019), and the faces this algorithm generates are indistinguishable from real people. It uses something called a generative adversarial network; basically, a machine learns to generate data that's similar to the data given to it (En.wikipedia.org, 2019) , and in this case, the data given was photographs.

    This one application of machine learning is going to lay the foundation for procedural generated virtual environments that are as life like as real life. Imagine putting on a VR headset and interacting with characters who were generated from generative adversarial networks. Likewise, the background itself was the result of a machine looking at landscapes and spitting out landscapes. Furthermore, if a machine can learn to generate images, it can surely learn to generate entire personalities and language patterns, creating artificial people with all the depth and complexity as human beings.

    The future of search engines will be profoundly accurate (Pickell, 2019) . Google is already using deep learning, a method of machine-learning that mimics the human brain (En.wikipedia.org, 2019), to provide highly personalised search results, but it's going to reach the point where it's so accurate, typing a handful of characters will come up with highly relevant suggestions, and these suggestions will be based on IoT connected devices like smart phones providing Google a wealth of personal data that they can use to further personalise search experiences.

    The future of software development could potentially be more about managing AI algorithms than teams of developers (Pickell, 2019). The idea of using an application to generate code instead of writing it by hand is not new (Mendix, 2019), and this idea is referred to as “low code”. What is ground breaking is the idea of a a machine learning to write code and producing an application, and this is referred to as “no code.” The irony of developers creating self-learning libraries that can be used to make themselves obsolete is fantastic.

    This idea, of course, isn't particularly substantiated. Machines are very good at doing repetitive tasks; however, we have one thing machines don't yet possess, and that's emotion. For this reason, a machine is unlikely to be able to produce artistic works with the same depth as human beings (The Drum, 2019); after all, art is an emotive medium.

    The media has a tendency to propagate fear. And in support of my previous point, 60 Minutes latched on to an AI expert claiming that AI would replace 40% of jobs (Inc.com, 2019). “It [leadership] requires a uniquely human quality that AI doesn't have: imagination. Without imagination, you can't put yourself in another person's shoes and sympathize with their point of view.“ The interview focused only on Lee's disconcerting statements and not on the empowering aspects of his work: that the age of AI will free humans to work creatively without the burden of processing.

    “AI will become one of the top five investment priorities for at least 30 percent of Chief Information Officers (DATAVERSITY, 2019) .” This trend in information technology will change the way information is processed. I anticipate a world where servers that users make requests to are nothing more than deep learning algorithms that generate computer code and return it to the user in the form of a website.

    What's currently known as an Application Programming Interface (API) may be later known as an Application Neural Interface (ANI). Likewise, user interfaces could potentially become adaptive in the sense that a user's artistic preferences may go on to shape the kind of UI they interact with. If Spotify can use machine learning to suggest songs a user may enjoy based on their existing song choices (Medium, 2019), an application could just as easily spawn an interface based on previous interactions with themes.

    Ultimately, I feel machine learning and its application as a sub field of AI will make a lot of data entry and processing jobs redundant, but it won't be as devastating as presented by the media. For example, Facebook has a team of content moderators that keep the platform clean of inappropriate content. You would think you could train an AI to identify inappropriate content and remove it, but it's not that simple (The Verge, 2019). Yes, an AI can identify a gun or nudity, but it's the context, which comes from contemporary culture, that dictate whether an old man's Winchester mounted over the mantle or a woman breastfeeding is different than a teenager showing off an unlicensed firearm or blatant pornography.

    In my daily life, I'm not certain I'll live to see the full fruition of this technology, but what it can already do is already improving my life through convenience. For example, Uber Eats uses AI to predict meal arrival times (Medium, 2019), and from personal experiences, it's very accurate. This is a simple example of how AI is already affecting me. It's not as exciting as creating an entire artificial landscape, but it is extremely practical.

    Furthermore, AI and machine-learning has made navigation a much easier experience (Agarwal, 2019). As someone who experiences driving related anxiety, having a navigation app that can correlate weather patterns, traffic data, accident reports, and use all of these data points to suggest an optimum route is an application of AI most aren't aware is happening.

    Finally, machine-learning is already extremely established in marketing to predict advertisements that will be relevant to the target audience (Kaput, 2019) , which is very nice. Ads are really only annoying when they're regarding products you don't care about, right. When I see an ad for a course on LinkedIn learning pop up on my feed based on courses I've already taken, and it's consequentially an interesting and useful subject to me, I genuinely don't mind, and I've used this AI -powered marketing in lieu of having to search for something myself, so having relevant information just handed to me is also a huge benefit.

    Clouds and Services go hand in hand, A Cloud is a Service that is accessible by the user on demand from the providers Servers.

    A Service is a system that provides the consumer with the resources that the user needs.

    There are three types of Cloud Computing, Software as a Service (SaaS), Platform as a Service (PaaS) and Infrastructure as a Service (IaaS).

    Software as a Service (SaaS) is a service model, in which the user pays for a software application, through the means of a subscription basis and then installed from the providers cloud datacenter. A Software as a Service is a month to monthly payment that the user pays to get access to the hardware or license. After the user has done the payment, the provider then delivers the software to your device over the internet. The software is streamed from the providers Servers, which means that the providers are responsible for maintaining the Servers and updating the software for the consumers. (Dummies, 2019).

    Platform as a Service (PaaS) is a cloud computing model, in which a provider, delivers software and hardware tools, for the user via the internet. A Platform as a Service (PaaS) allows the consumer to operate applications without having to first install the hardware and or software required. A Platform as a Service (PaaS) provider, builds and supplies the consumer with a strong and advanced environment, that the user can then use to install applications and data sets. This allows the consumers to focus on creating and running the applications, rather than constructing and maintaining the framework and services themselves. Due to the feature of a PaaS, a business will usually rely on PaaS providers for their key services. For example, java development and application hosting. (Dummies, 2019).

    Infrastructure as a Service (IaaS) is the process of moving the core infrastructure equipment to the Cloud. This allows companies and businesses to no longer need to worry about buying and maintaining said equipment. These equipment’s can be anything from Telephone systems, to Servers and Firewalls. Infrastructure as a Service (IaaS) allows the consumers to purchase the services like a utility for the equipment that is required. IaaS allows businesses to save money, because It saves the consumer from having to hire a specialist to come around to the location of their company and go around the infrastructure repairing or improving the equipment, this saves money because the consumer will not have to hire the person, because all the equipment will be in one place in the Cloud. However due to this, the rolls of technicians are at risk. (Dummies, 2019).

Any computer that is not a Server are referred to as a Client. A Server is usually the most expensive and powerful computers in a network, where the less powerful and cheaper computers in a network are the Clients. The Clients are the computers that individual users use for everyday work. Due to the functionality of a Client, a network will always have more Clients then Servers, for example, a network with ten Clients, could probably get by with just the one Server.

    Mod Manager Overlay for Twitch

    The project we have chosen is based on Jamie's idea for Assignment 1 (Walters, 2019). Jamie's original proposal outlines a graphical interface that allows game streamers on the Twitch platform the ability to add, remove, and find mods for the game they are playing without having to switch to a third-party tool in the middle of a live stream.

    Twitch can be used to broadcast video streams by using their website, but they also have a dedicated application for Windows, MacOS, Android, and iOS (Twitch, 2019). The mechanism through which our mod-manager will interface between Twitch, local game files, and hosted mods is called Twitch Extensions (Twitch Developers, 2019). Extensions expand the base functionality of streams by adding interactive UI elements to a variety of views from the video player itself to the streamer's profile page, leading to higher audience engagement and a more personalised experience.

    Technical Overview

    In creating a Twitch extension through the developer portal, the developer is required to choose where their extension will appear on the Twitch UI. Our extension will appear as an overlay over the streamed video in both docked and full screen mode. Viewers will be able to see the mods the streamer is using in the current game, but only the streamer will be able to modify said mods.



    After developers choose their extension type, they are prompted to download a developer tool called Twitch Developer Rig. This tool allows developers to test extensions in a local environment using their Electron-wrapped desktop application (Twitch Developers, 2019).

    The below image shows manual configuration details and environmental variables that Twitch Developer Rig will automatically configure for ease of use.



    This GUI tool basically plugs into a folder that contains the local files for the extension, and when a developer clicks “Run Front End” from within the Developer Rig, the GUI will simply run the corresponding script from package.json.

    The below image shows a folder structure for a working test extension to give developers a head start in extension development.

    These files were automatically generated by Twitch Development Rig



    The biggest advantage of using Twitch Developer Rig is being able to see how our extension looks inside a simulated Twitch UI.

    The below image reflects how our overlay might appear over the Twitch video and allows us to visualize how our application works.



    The architecture of this application is as follows. When the streamer configures the extension, a node.js server is installed at the root of the streamer's Steam games folder (Support.steampowered.com, 2019).

    When the streamer interacts with the overlay over their Twitch stream, their client will send a request to the Node server, which will in turn update the local files inside its running directory.

    The current state of the application will be sent to another public server that will be the official back end of this application. Because the server is public, viewers will be able to see the mods a streamer is running.

    Modifying mods will not be accessible to viewers, and this view will be hidden behind a configuration view that's only accessible to the streamer.

    The mods themselves are all hosted on the public server, and in addition to this application being a highly convenient tool for Twitch, this platform will serve as an ecosystem to publish and host mods in a way where they're accessible for both streamers and their viewers, leading to increased discovery for the developers who create these mods and the gamers who use them.

    Features

    • Clean easy to use overlay UI.
    • Add-on/mods finder.
    • Social tab to talk to friends not in game.
    • Forum tab to see what people are saying about the game you are in.
    • Creators pages, to submit bugs or to provide feedback.

    I would like the UI to be clean and easy to use. It would need to be slightly transparent as to still see what is going on in-game. At the top you would have tabs going from left to right displaying the different categories such as home, add-ons, social, forums, feedback and help.

    The home tab would just have an overview of all recent activity. The colour theme would still be the twitch purple but a little transparent to make it look more of a lavender colour.

    The add-ons tab would have your usual search, browse, top add-ons trending and paid if they wanted to go down that route. If you are using this overlay, you would most likely know what add-ons your looking for so having the search button first would just be easier.

    The social tab would have your friends online and who is offline, you will be able to click onto your friend’s name and talk to them via private message. You will also be able to view your friend’s profile as well. The forum tab will be simple with a search button and a create button, it would also have a trending and new part of the page with about the 10 of each on the page.

    The last tab would be where you search and see creator’s pages that have all their add-ons so you can provide feedback or even submit bugs/problems with the addons they are providing. You will be able to activate the overlay via a keystroke of your choosing.

    Feasibility

    Twitch allows developers to host their front end files and extension files for free; however, our back end will need a dedicated server and a database.

    Additionally, a cross-platform installer that can detect the location of a streamer's local game files and copy a Node server to this location will be necessary.

    The Twitch EBS (Extension Backend Service) is the biggest technical obstacle in building this application. A detailed knowledge of the Twitch API is necessary to connect an application's front end to a backend server that's interfacing with Twitch.

    A developer working on this project will need to have substantial backend experience with Node.js and deployment experience with Twitch extensions.

    Likewise, this developer will need a working understanding of React.js as well as Webpack build tools.

    Overall, a mid-level to senior engineer, or a junior engineer with a decent amount of experience using EBS, will be required.

    Because this application is also a platform for discovering mods, this project could benefit from a team of at least three developers.

    I see this application as having three parts:

    • The node server connecting to the streamer's local game files
    • The frontend and backend of the extension reflecting a streamers mods
    • The platform for discovering and publishing new mods

    It would therefore make the most sense to break this project up into three parts and have a dedicated developer for each part.

    Regarding the entry requirements for users, any user capable of running a web browser is able to use this application.

    Use Case

    There is currently no mod management tool for Steam-based games that also interfaces with Twitch.

    There is nothing worse than having to exit a game completely to add a mod or add-on. Most add-ons are just UI changes so if you just reload the UI in World of Warcraft for example you should be able to see the new add-on you just installed. Just like a smartphone, installing applications does not require to restart the phone. So just a little add-on to a game should not require you to close the game completely.

    Not only is switching tabs mid stream inconvenient, it's also risky.

    Every time a streamer has to alt-tab out of their game while streaming live, they risk revealing sensitive information to their audience and doxxing themselves (The Conversation, 2019).

    I believe this extension would be a very beneficial addition to an already growing company and its users.

    Being able to add mods on the fly, talk to content creators, provide feedback, and also search forums all without alt tabbing out of the game. I believe if it was to be created it would have a welcoming reception to the users of Twitch and Curse.
